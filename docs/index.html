Helm Repository

for AI Systems on K3s in Conformity to ISO/IEC 42001

Structured as a 4-tier reference architecture mapping Device, Edge, Platform, and Enterprise layers to 18 ISO/IEC 42001 Annex B requirements. Each solution includes a Helm chart, Rancher deployment questionnaire, and traceability to specific compliance clauses.

Designed for Rancher and K3s with Traefik ingress, local-path storage, and lightweight resource profiles. Add this catalog to Rancher to deploy, configure, and manage each component via the UI with guided questionnaires for every deployment variable.

Architecture Overview

Enterprise Tier

Governance, identity management, and incident response

keycloak zammad minio-docs grafana-dashboards

Platform Tier

AI lifecycle management, centralised monitoring, and data warehousing

mlflow training-jobs prometheus grafana loki minio timescaledb postgresql rancher

Edge Tier

Data ingestion, real-time inference, and local monitoring at the plant floor

node-red kafka mosquitto fastapi-model fluent-bit prometheus-agent falco postgresql mongodb

Quick Start

terminal
# Clone the catalog
git clone https://github.com/CIGIP-UPV/MLOps-ISO42001-K3s-Catalog.git
cd MLOps-ISO42001-K3s-Catalog

# Prepare Ubuntu 24.04 (installs K3s, Helm, Rancher CLI)
sudo ./infrastructure/setup-ubuntu.sh --server

# Deploy the full stack
./infrastructure/install.sh --all

# Or deploy a single tier
./infrastructure/install.sh --edge-only
./infrastructure/install.sh --platform-only
./infrastructure/install.sh --enterprise-only

# Deploy an individual chart
cd catalog/edge/storage/postgresql/manifests
helm dependency update .
helm install postgresql-edge . -n edge --create-namespace
